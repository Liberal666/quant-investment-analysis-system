# 基于 Spring Boot + Vue3 的量化投资绩效分析系统项目报告

## 摘要

本项目设计并实现了一个前后端分离的量化投资绩效分析系统。系统以后端 Spring Boot 为核心，结合 MyBatis 和 MySQL 完成股票 K 线数据存储、技术指标计算、相关性分析、策略记录、用户权限管理、登录注册与 AI 辅助分析；前端采用 Vue3、Axios 和 ECharts 构建可视化页面，支持 K 线图、均线、MACD、RSI、收益曲线、相关性散点图和多股票区间收益报表展示。

系统数据主要来源于新浪财经非官方接口，并设计了腾讯行情接口与本地 Mock 数据作为兜底方案，保证在行情接口临时不可用时系统仍可运行和演示。项目同时提供真实数据重置脚本，可向数据库中写入近一年真实行情数据，并对非交易日或缺失日期使用前一交易日数据补齐，满足课程设计展示和答辩需要。

关键词：Spring Boot；Vue3；MyBatis；MySQL；ECharts；量化投资；技术指标；相关性分析

## 1. 项目背景与意义

随着证券市场数据化程度不断提高，投资分析已经逐渐从单纯依赖人工经验转向数据驱动和模型辅助决策。股票历史价格、成交量、技术指标、收益波动、市场相关性等信息，能够帮助投资者更直观地了解个股表现与风险特征。

本系统以课程设计为目标，围绕“数据获取、指标计算、绩效分析、风险展示、AI 辅助解释”构建完整的量化投资绩效分析流程。系统既能展示真实股票行情数据，也能通过可视化图表呈现分析结果，具有较强的工程完整性和答辩展示价值。

## 2. 需求分析

### 2.1 功能需求

系统主要功能如下：

1. 用户登录与注册  
   用户可通过账号密码登录系统，新用户可注册为普通用户。登录注册页面参考股票投资平台视觉风格设计，包含金融行情背景、指数行情、K 线元素和登录/注册切换功能。系统包含管理员和普通用户两类角色。

2. 股票数据管理  
   支持按照股票代码查询股票历史 K 线数据，包括开盘价、最高价、最低价、收盘价和成交量。支持将股票数据同步到 MySQL 数据库。

3. 技术指标计算  
   后端使用 Java 实现 MA、MACD、RSI 等技术指标计算，并将计算结果存入数据库。

4. 相关性分析  
   使用 Pearson 相关系数计算个股与比较基准之间的收益相关性，默认支持沪深 300 指数作为比较基准。

5. AI 辅助分析  
   后端调用 DeepSeek OpenAI 兼容接口生成自然语言分析。当外部 AI 接口不可用时，系统返回本地规则生成的分析内容，保证可运行。风险分析页面还支持基于股票代码、行情上下文和用户问题的 AI 问答。

6. 前端可视化展示  
   使用 ECharts 展示基金收益、净值变动、相关性散点图、K 线与 MA 均线、MACD、RSI、多股票区间收益等图表。

7. 策略管理  
   用户可以录入交易日期、股票代码、策略类型和策略内容，系统将策略记录持久化保存到数据库。

8. 风险分析  
   用户输入股票代码后，可查看实时行情信息、单只股票 K 线图、MACD 图和 AI 参考建议。

9. 数据报表  
   支持选择一只或多只股票，设置开始日期和截止日期，计算区间收益、成交量和数据条数，并进行图表展示。

10. 权限管理  
    普通用户只能访问数据，管理员可以额外管理用户权限。

### 2.2 非功能需求

1. 可运行性  
   系统应包含基础数据与 Mock 兜底方案，避免行情接口失败导致项目无法展示。

2. 可维护性  
   后端采用 controller、service、mapper、entity、util 分层结构，前端采用 views、components、api 结构，便于维护。

3. 可扩展性  
   股票数据接口、AI 分析接口和比较基准设计为可扩展形式，后续可替换为正式行情服务或更多 AI 模型。

4. 展示友好性  
   前端页面采用股票行情终端风格，登录页复刻金融投资网站风格，主页面以深色行情终端为主，适合课程设计演示和答辩。

## 6. 核心模块设计

### 6.1 登录注册模块

登录注册模块由 `AuthController`、`UserService`、`UserMapper` 和前端 `LoginView.vue` 实现。用户输入用户名和密码后，前端通过 Axios 调用 `/api/auth/login`；注册时调用 `/api/auth/register`。注册用户默认拥有普通用户权限，并绑定基础股票，避免首次进入系统无数据。

密码不以明文存储，而是保存 SHA-256 哈希值。用户登录成功后，系统返回用户信息，并进入主系统页面。登录页面包含“智投财经 SMART INVEST”标识、指数行情、城市天际线、K 线柱、上升箭头、登录/注册 Tab、记住我和忘记密码入口。按照当前课程设计要求，页面不提供微信、QQ、手机号短信等第三方登录方式，仅保留系统账号密码登录与注册。

### 6.2 股票数据模块

股票数据模块通过 `StockController` 提供 API，由 `StockServiceImpl` 完成业务逻辑。系统支持：

- 查询股票 K 线数据
- 同步股票数据到数据库
- 查询实时行情
- 添加用户自选股票
- 查询产品净值列表

后端使用 `SinaFinanceClient` 调用新浪财经接口获取历史 K 线和实时行情。当新浪实时行情失败时，系统尝试腾讯行情接口；若外部接口不可用，则使用数据库数据或 Mock 数据进行兜底。

### 6.3 技术指标计算模块

技术指标计算由 `IndicatorCalculator` 实现。系统支持：

- MA5
- MA10
- MA20
- MACD
- RSI14

指标计算在后端 Java 中完成，计算结果写入 `stock_indicator` 表。

### 6.4 相关性分析模块

相关性分析由 `CorrelationCalculator` 实现。系统计算个股收益率序列与比较基准收益率序列之间的 Pearson 相关系数，并返回散点图数据。

### 6.5 AI 辅助分析模块

AI 辅助分析由 `DeepSeekClient` 实现。系统将股票表现、相关性结果等信息发送给 DeepSeek API，并返回自然语言投资解读。当 API 未配置或请求失败时，系统返回本地规则生成的分析内容。

系统还提供 `/api/stock/chat` 问答接口。前端风险分析页面可将股票代码、当前行情上下文和用户问题提交给后端，由后端组织 Prompt 并调用 DeepSeek，返回更贴近当前股票走势的参考建议。

### 6.6 策略管理模块

策略管理模块由 `TradeStrategyController`、`TradeStrategyService` 和 `TradeStrategyMapper` 实现。用户可以录入交易策略，系统将策略日期、股票代码、策略类型和策略内容保存到数据库。

### 6.7 权限管理模块

权限管理模块由 `UserController` 和 `UserServiceImpl` 实现。管理员可以修改用户角色、数据访问权限和用户管理权限。普通用户只能查看数据，不能管理其他用户权限。

### 6.8 前端可视化模块

前端使用 ECharts 进行图表展示，主要图表包括：

- 基金收益折线图
- 净值变动图
- 相关性散点图
- K 线与 MA 均线图
- MACD 指标图
- RSI 指标图
- 多股票综合收益图

## 7. 核心算法说明

### 7.1 MA 均线

MA 是移动平均线，用于反映一段时间内股票收盘价的平均水平。

```text
MA(N) = 最近 N 日收盘价之和 / N
```

系统实现了 MA5、MA10 和 MA20，用于观察短期和中期趋势。

### 7.2 MACD 指标

MACD 用于判断价格趋势和动能变化。系统计算过程如下：

```text
EMA12 = 12 日指数移动平均
EMA26 = 26 日指数移动平均
DIF = EMA12 - EMA26
DEA = DIF 的 9 日指数移动平均
MACD = (DIF - DEA) * 2
```

当 MACD 柱持续扩大时，通常表示趋势动能增强；当 MACD 柱持续缩小时，表示趋势动能减弱。

### 7.3 RSI 指标

RSI 用于衡量价格上涨和下跌力量对比。本系统计算 RSI14：

```text
RS = 14 日平均上涨幅度 / 14 日平均下跌幅度
RSI = 100 - 100 / (1 + RS)
```

RSI 较高通常表示价格偏强，RSI 较低通常表示价格偏弱。

### 7.4 Pearson 相关系数

Pearson 相关系数用于衡量两个收益率序列之间的线性相关程度。

```text
r = Σ((xi - x平均值)(yi - y平均值)) / sqrt(Σ(xi - x平均值)^2 * Σ(yi - y平均值)^2)
```

相关系数取值范围为 `[-1, 1]`：

- 接近 1：高度正相关
- 接近 0：相关性较弱
- 接近 -1：高度负相关

## 10. 数据初始化与真实数据导入

项目提供 `database/reset_real_data.py` 脚本，用于初始化真实股票数据。脚本功能如下：

1. 创建或保留数据库表结构。
2. 清空旧业务数据，但保留字段结构。
3. 从新浪财经接口获取近一年真实股票日 K 数据。
4. 对非交易日或缺失日期复制前一个可用交易日数据。
5. 计算并写入 MA、MACD、RSI 指标。
6. 初始化普通用户和管理员账号。
7. 为用户绑定至少 10 只股票。

当前脚本包含以下股票：

```text
000001 平安银行

000063 中兴通讯

000858 五粮液
002415 海康威视
300750 宁德时代
600036 招商银行
600519 贵州茅台
```

同时包含 `000300` 沪深 300 作为比较基准。

## 13. 项目特色

1. 数据链路完整  
   系统包含外部行情获取、数据库存储、指标计算、前端可视化的完整链路。

2. 指标逻辑由 Java 实现  
   MA、MACD、RSI、Pearson 相关系数均由后端 Java 实现，符合课程设计对核心计算逻辑的要求。

3. 前端可视化完整  
   使用 ECharts 展示 K 线、均线、MACD、RSI、相关性散点图和多股票收益图。

4. 真实数据与 Mock 兜底并存  
   系统既支持真实行情数据，又在接口失败时提供兜底数据，保证演示稳定性。

5. 包含权限管理和策略管理  
   系统不仅展示行情分析，还具备用户权限和策略记录等业务功能。

6. 支持 AI 辅助解读  
   系统可调用 DeepSeek API，将数值分析结果转化为自然语言解释，并支持风险分析页面中的 AI 问答。

7. 登录页面具有金融平台辨识度  
   登录页复刻股票投资网站布局，包含行情视觉、指数数据、K 线元素和深蓝登录卡片，同时删除第三方登录入口，更符合当前系统账号体系。

## 14. 总结与展望

本项目完成了一个基于 Spring Boot 和 Vue3 的量化投资绩效分析系统，具备股票数据获取、技术指标计算、相关性分析、风险展示、策略管理、权限管理、数据报表和 AI 辅助分析等功能。系统结构清晰，前后端分离明确，数据库设计较完整，适合作为课程设计项目进行展示。

后续可从以下方向继续完善：

1. 引入正式证券行情数据源，提高数据稳定性和准确性。
2. 增加更多技术指标，如 Bollinger Bands、KDJ、ATR 等。
3. 增加回测模块，支持策略收益、最大回撤、夏普比率等系统化评估。
4. 引入 JWT 或 Spring Security，提高用户认证安全性。
5. 支持组合管理，实现多股票持仓权重和组合收益分析。
6. 增加导出报表功能，将分析结果导出为 Excel 或 PDF。
